
Проект "Байкал Без Пластика"


Это мобильное приложение, которое помогает распознавать типы пластика по фотографии.  
Пользователь фотографирует упаковку или предмет из пластика, нейросеть определяет тип (ПЭТ, ПНД, ПВХ и т.д.), выдаёт информацию о переработке и начисляет баллы.  
Задумка — повысить осведомлённость о сортировке отходов и мотивировать людей правильно утилизировать пластик.

Технологии:
- Клиент: React Native (Expo) + React Navigation + AsyncStorage + камера.
- Сервер: FastAPI (Python), SQLAlchemy (async), PostgreSQL, bcrypt, JWT.
- Машинное обучение: TensorFlow/Keras, модель классификации пластика по изображениям.
- Хранение: PostgreSQL, загруженные аватары хранятся в папке "uploads/avatars".

Ниже подробно описан весь процесс установки и запуска.

2. Установка и запуск сервера (FastAPI + PostgreSQL)
--------------------------------------------------------------------------------
2.1. Установите PostgreSQL (например, версию 14 или выше).  
     Во время установки запомните пароль для суперпользователя postgres.  
     Создайте базу данных. Можно через командную строку:
        psql -U postgres -c "CREATE DATABASE baikalbp;"

2.2. Перейдите в папку backend:
        cd backend

2.3. Создайте виртуальное окружение Python:
        python -m venv venv

     Активируйте его:
        Windows:   venv\Scripts\activate
        Mac/Linux: source venv/bin/activate

2.4. Установите все зависимости из requirements.txt:
        pip install -r requirements.txt

     Если нет pip, сначала установите: python -m pip install --upgrade pip.

2.5. Настройте переменные окружения в файле .env (он уже есть в папке backend).  
     Пример содержимого:
        DATABASE_URL=postgresql://postgres:ваш_пароль@localhost/baikalbp
        SECRET_KEY=dreamworsk   # можно сгенерировать что-то сложное
        ALGORITHM=HS256
        ACCESS_TOKEN_EXPIRE_MINUTES=1440

     Убедитесь, что в строке DATABASE_URL указаны правильные логин, пароль и имя базы.

2.6. Запустите сервер разработки:
        uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

     Если всё прошло успешно, вы увидите сообщение: "INFO:     Uvicorn running on http://0.0.0.0:8000".

2.7. Проверьте, что API доступен: откройте http://localhost:8000/docs – там должна быть интерактивная документация Swagger.

Примечания:
   - Таблицы в базе данных создадутся автоматически при первом запросе (благодаря "Base.metadata.create_all" в startup).
   - Папка "uploads/avatars" создаётся автоматически при загрузке аватара.

--------------------------------------------------------------------------------
3. Запуск клиента (Expo)
--------------------------------------------------------------------------------
3.1. Убедитесь, что у вас установлен Node.js (версия 16 или выше). Проверьте:
        node -v

3.2. В корне проекта выполните:
        npm install

     Это установит все зависимости, перечисленные в package.json (expo, react-navigation, axios, etc).

3.3. Откройте файл src/config.js и укажите правильный адрес вашего сервера.  
     Например:
        export const API_BASE_URL = 'http://localhost:8000';
     Если сервер запущен на другом компьютере, укажите его локальный IP (например, 192.168.1.10).

3.4. Запустите Expo:
        npx expo start

     В терминале появится QR-код. Скачайте на смартфон приложение Expo Go (iOS/Android) и отсканируйте QR-код, либо нажмите клавишу "a" для Android-эмулятора или "i" для iOS-симулятора.

3.5. После загрузки приложения вы увидите экран входа.  
     - Для регистрации нажмите "Нет аккаунта? Зарегистрироваться".  
     - Введите имя, email и пароль (пароль хешируется на сервере).  
     - После регистрации можно будет загрузить аватарку и выбрать отображаемое имя.  
     - На главном экране есть кнопка камеры – сделайте фото пластика, дождитесь распознавания.  
     - Если тип пластика определён, вам начислят 1 балл (отражается в профиле).

--------------------------------------------------------------------------------
4. Особенности и возможные проблемы
--------------------------------------------------------------------------------
4.1. CORS: бэкенд уже настроен на приём запросов с любых источников (через CORS middleware).  
     Если нужно ограничить, поправьте в "app/main.py" параметр "allow_origins".

4.2. Проблемы с камерой на Android: в app.json уже прописаны разрешения "android.permission.CAMERA".  
     На реальном устройстве разрешение запросится при первом использовании.

4.3. Модель не загружается: проверьте, что файлы "plastic_classifier.h5" и "class_names.txt" лежат в папке "backend/app/models/".  
     Если их нет, скопируйте из архива или создайте заново (учебная модель в репозитории уже есть).

4.4. Ошибка "ModuleNotFoundError: No module named 'asyncpg'": установите "pip install asyncpg".  
     В requirements.txt он есть, но иногда нужно установить отдельно.

4.5. При регистрации появляется ошибка "username or email already registered" – значит, такой пользователь уже существует.

4.6. База данных не создаётся: проверьте, что PostgreSQL запущен и доступен по адресу, указанному в DATABASE_URL.  
     В Windows можно проверить через Services.msc, на Mac/Linux – "sudo systemctl status postgresql".

4.7. Аватарка не загружается: убедитесь, что папка "uploads/avatars" существует и имеет права на запись (на Linux может понадобиться "chmod 777 uploads/avatars").  
     В Windows права обычно выставляются автоматически.

4.8. При сканировании изображения возвращается "Internal server error": проверьте логи uvicorn. Чаще всего проблема в неправильной модели или недостаточных правах на папку с моделью.

--------------------------------------------------------------------------------
5. Работа с моделью машинного обучения
--------------------------------------------------------------------------------
Модель "plastic_classifier.h5" обучена на датасете из папки "dataset/".  
Классы: HDPE, LDPE, OTHER, PET, PP, PS, PVC.  
При распознавании изображения возвращается название класса и уверенность.  
Если класс определён, клиент получает дополнительную информацию (описание, срок разложения, советы по переработке) из "plastic_data.py".  
За каждое корректное сканирование (класс из списка пластиковых) начисляется 1 балл.

Модель загружается один раз при старте бэкенда. Если вы хотите обучить свою модель, можете заменить файл "plastic_classifier.h5" и обновить "class_names.txt".

--------------------------------------------------------------------------------
6. Как использовать
--------------------------------------------------------------------------------
- После запуска приложения зарегистрируйтесь (или войдите, если уже есть аккаунт).  
- Выберите отображаемое имя (можно пропустить).  
- На главном экране нажмите кнопку камеры, сфотографируйте пластиковый предмет.  
- Дождитесь результата – приложение покажет название пластика, интересные факты и начислит балл.  
- В профиле можно посмотреть баланс, изменить аватарку и имя.  
- Все данные хранятся на сервере, но токен сохраняется на устройстве, чтобы не входить заново.

--------------------------------------------------------------------------------
7. Что делать, если что-то работает не так
--------------------------------------------------------------------------------
- Не работает вход: проверьте, что сервер запущен и порт 8000 открыт.  
- Не распознаётся пластик: возможно, модель не загрузилась – загляните в логи бэкенда.  
- Баллы не начисляются: убедитесь, что объект классифицирован как пластик (один из известных классов).  
- Аватарка не показывается: URL аватара генерируется как "/static/avatars/имя_файла".  
  В браузере можно открыть http://localhost:8000/static/avatars/имя_файла – если не отображается, проверьте права.
